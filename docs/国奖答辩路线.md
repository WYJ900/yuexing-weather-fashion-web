# 悦行天气国奖答辩路线

## 1. 作品定位

悦行天气是一款“天气与情绪融合的智能行程助手”。

它解决的不是单点穿搭推荐，而是日常出行中的连续问题：出门前不知道天气风险、心情状态和行程安排如何共同影响穿搭与携带物；出门后体验容易被遗忘，系统也无法从真实反馈中学习。

项目因此设计为一个闭环系统：

出门前生成方案 → 出门中提供提醒 → 出门后记录手记 → 系统学习画像 → 下一次推荐更个性化。

## 2. 10 分钟演示结构

### 0:00 - 1:00 问题与定位

说明学生和年轻用户的日常痛点：

- 天气变化影响衣物和携带物。
- 情绪状态会影响对舒适、正式、轻便的需求。
- 出行经验常被遗忘，下一次仍然重复踩坑。

一句话收束：

悦行天气让天气、情绪、行程和记忆一起参与推荐。

### 1:00 - 2:20 首页与系统架构

打开首页，强调：

- 真实天气接入。
- 情绪语义理解。
- 行程场景建模。
- 个人衣柜画像。
- 行程手记记忆。

### 2:20 - 4:20 今日行程

点击“今日行程”前先展示登录逻辑：

- 系统首次访问默认进入登录页。
- 支持真实注册和登录，也保留演示体验账号 `demo / Yuexing@2026`。
- 说明衣柜、手记和画像都是个人数据，不登录不能进入核心工作台。

登录后进入“今日行程”：

- 展示 Open-Meteo 天气来源。
- 输入一句状态，例如“今天有点累，但下午要去面试，希望轻便又稳重”。
- 选择行程场景、交通和在外时长。
- 展示行程方案、携带清单、风险提醒指数、舒适匹配度。
- 强调可解释推荐链：天气依据、情绪依据、衣柜依据、手记依据。

### 4:20 - 5:10 我的衣柜

进入“我的衣柜”：

- 展示分类衣柜。
- 说明上传衣物图片后可提取主色和风格标签。
- 强调衣柜不是静态展示，而是推荐系统的个人物品库。

### 5:10 - 7:00 我的手记

进入“我的手记”：

- 新增一条手记：地点、日期、情绪、交通、照片、正文。
- 保存后展示 AI 摘要、标签、情绪复盘和下次建议。
- 说明照片主色、正文关键词、天气快照都会进入画像学习。

建议演示文本：

“今天去湖边散步，多云不晒，心情很松弛。走了很久有点累，幸好穿了舒服的鞋。下次要少带东西。”

预期标签：

- 自然疗愈
- 低负担
- 舒适鞋
- 多云
- 步行

### 7:00 - 8:10 月度 / 年度总结

切换月度总结和年度总结：

- 展示出行次数。
- 展示常去地点、常见天气、常见情绪、常用交通和关键词。
- 强调年度总结不是普通表格，而是“你走过的天气”的长期陪伴表达。

### 8:10 - 9:10 个性画像

进入“个性画像”：

- 点击“根据手记重建画像”。
- 展示常去地点、喜欢天气、情绪模式、出行方式、行程类型、记忆关键词。
- 说明画像会回流到今日行程，影响提前时间、携带清单、舒适鞋和低负担路线。

### 9:10 - 10:00 技术总结与创新点

收束到作品说明页，强调：

- 真实数据：Open-Meteo。
- 多模态输入：文本、衣物图片、行程照片。
- 智能决策：规则 + LLM + 画像学习。
- 可解释性：每条建议都有依据。
- 持续学习：手记和保存方案共同更新画像。
- 稳定性：AI 不可用时使用本地 fallback。

## 3. 答辩关键词

- 不是天气查询，而是行程决策。
- 不是一次性推荐，而是长期学习。
- 不是黑盒生成，而是可解释推荐。
- 不是纯页面展示，而是有 SQLite 持久化和 API 的完整原型。
- 不是依赖外部模型炫技，而是把 AI 工具组合进真实生活问题。

## 4. 评委可能追问

### AI 具体在哪里？

AI 体现在四层：

- 文本语义：一句话状态和手记正文提取情绪与关键词。
- 图像理解：衣物和手记照片提取主色，辅助风格和场景标签。
- 决策推荐：天气、行程、衣柜和画像共同生成方案。
- 生成总结：手记摘要、情绪复盘、月度/年度总结。

### 如果大模型不可用怎么办？

系统设计了本地 fallback：

- 情绪识别可由关键词规则完成。
- 手记标签可由本地语义规则生成。
- 行程建议和总结可用模板生成。
- 现场演示不会因为 API 波动中断。

### 为什么属于人工智能实践赛？

项目针对日常出行这一具体领域问题，提出 AI 化解决方案，并实现了真实数据接入、语义分析、图像主色提取、个性画像、推荐解释、手记复盘和持续学习闭环，具备完整方案设计、代码实现和功能演示。

### 创新点是什么？

创新点在于把“天气推荐”升级为“天气 + 情绪 + 衣柜 + 手记记忆”的长期行程助手。系统能从用户真实出行后的文字和照片中学习偏好，让下一次出门建议更贴近用户。

## 5. 提交前检查清单

- 首页定位是否已改为智能行程助手。
- 今日行程能否展示真实天气或 fallback。
- 手记能否新增并生成摘要。
- 月度/年度总结是否有内容。
- 个性画像能否重建。
- README、项目规划、答辩文档是否一致。
- 演示视频是否控制在 10 分钟内。
