# 悦行天气部署上线说明

## 1. 推荐部署方式

当前项目最适合用 Render 以单个 Node Web Service 方式部署。

原因：

- 前端 `npm run build` 后产物位于 `dist/`
- `server.js` 已支持直接托管 `dist`
- 同一个服务同时负责静态页面和 `/api/*`
- DeepSeek key 只保留在服务端环境变量
- 项目根目录已经提供 `render.yaml`

## 2. Render 配置

### 基础信息

- Runtime: `Node`
- Build Command: `npm install && npm run build`
- Start Command: `npm start`

### 环境变量

- `AI_BASE_URL=https://api.deepseek.com/v1`
- `AI_API_KEY=<你的 DeepSeek Key>`
- `AI_MODEL=deepseek-chat`

可选：

- `PORT`
  Render 会自动注入，通常不用手动填写

## 3. 本地确认项

上线前建议先确认：

- `.env` 已配置真实 DeepSeek key
- `npm run dev:full` 能正常打开
- 注册、登录、退出登录能跑通
- 今日行程能看到 `ai-online` 或可接受的 `local-fallback`

## 4. Render 上线步骤

1. 将项目推到 GitHub 仓库
2. 登录 Render，创建 `Web Service`
3. 连接 GitHub 仓库，或直接导入仓库中的 `render.yaml`
4. 填入构建命令和启动命令
5. 配置环境变量
6. 触发首次部署
7. 部署成功后获得 `https://<service>.onrender.com`

## 4.1 健康检查

服务端已提供：

- `GET /api/health`

Render 可直接用它做启动后健康检查。

## 5. 域名

如果你有自己的域名，可以在 Render 中继续绑定自定义域名。

参考：

- [Render Web Services](https://render.com/docs/web-services)
- [Render Environment Variables](https://render.com/docs/configure-environment-variables)
- [Render Custom Domains](https://render.com/docs/custom-domains)

## 6. 当前上线边界

- SQLite 目前随服务一起运行，适合原型和小规模演示
- 若后续要做长期线上使用，建议把图片迁移到对象存储，把数据库迁移到托管数据库
- 若要支持真正多端同步，衣柜和偏好也建议迁移到后端持久化


## 2026-04-24 安全发布补充

- 登录态已切换为 HttpOnly Cookie，会话由服务端签发和校验；部署后前端无需保存 token。
- Render 环境下建议保持同域部署，避免额外跨域复杂度；如需单独放开来源，可配置 `CORS_ORIGINS`。
- 当前默认请求体限制为 2.5MB，手记图片限制为 1.5MB；上线前请提醒用户压缩大图。
- 手记图片会保存到服务端 `/uploads/*`。Render 免费实例适合演示与小规模使用，长期运行建议迁移到对象存储。
